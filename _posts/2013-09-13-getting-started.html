Getting started with Vessel

Step 1: Download

  1. Login to Vessel Command Center
  2. Add your application
  3. Get Vessel Secret Key as well as Download Vessel SDK.

Step 2: Install

  1. Unzip the downloaded vesselsdk.zip
  2. Import the vesselsdk.jar into your existing Android project, by extracting it into your "libs" folder.
  3. If your project does not already have a "libs" folder, create one at the root of the project by right-clicking the project and choosing "New" and then "Folder.

Step 5 (Optional)

In case if you are using ProGuard for your application, then add following lines in your proguard.cfg file, It will be located inside your project folder.It will exclude VesselSDK from obfuscation process.

{% highlight java %} -dontwarn com.vessel.* -dontnote com.vessel.* -keep class com.vessel.** { *; } {% endhighlight %}

At this point you have successfully integrated VesselSDK in your application , to test the integration build and install application on your test device or on simulator. Proceed to next tutorial, enable in-app support desk.

Getting started with Vessel

Step 1: Download

  1. Login to Vessel Command Center
  2. Add your application
  3. Get Vessel Secret Key as well as Download Vessel Framework.

Step 2: Install

  1. Make sure you are using the latest version of Xcode (v4.6+) and targeting iOS 4.3 or higher.
  2. Unzip the file you downloaded and drag the Vessel.framework folder into your Xcode project folder target.
  3. Make sure the 'Copy items to destination's group folder' checkbox is checked and the target to be added to is selected.

Step 3: Integrate

Make sure you include the Vessel library headers in every file where you use Vessel. Initialize Vessel as below, preferably in the beginning of your application delegates application:didFinishLaunchingWithOptions:

{% highlight obj-c %} #import - (BOOL) application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions { [Vessel initializeWithAppSecret:@"Vessel app key"]; [VesselCrashReporter enableCrashReporting] } {% endhighlight %}

Step 4: Enable Server Side Symbolification

Vessel supports server side symbolication. This allows the debug symbols to be stripped from the binary making the binary size smaller. For server side symbolication to work properly, you need to upload the dSYM file of the build. This can be automated by adding following script to the 'Build Phases'.

  1. Click on your target.
  2. Click on the 'Build Phases' tab.
  3. Click and hold down your left mouse button on 'Add Build Phase' button at the bottom right corner of the window.
  4. In the menu that appears, click on 'Add Run Script'.
  5. 'Run Script' panel gets added to the 'Build Phases'. Make sure that the 'Run Script' phase is the last one in the 'Build Phases'. You can rearrange the phases by drag-dropping the build phase panels.
  6. Copy following string in the text box below the text box that containing text / bin/sh!
    7. {% highlight obj-c %} ./Vessel.framework/Resources/upload.sh YOUR_APP_SECRET_KEY {% endhighlight %}

Step 5:

You will need to link 'SystemConfiguration.framework' & 'AdSupport.framework' (for iOS7) with your target. If you already linked these two frameworks, you can skip this step.

  1. Click on your target.
  2. Click on the 'Build Phases' tab.
  3. Expand the 'Link Binary With Libraries' panel.
  4. Click on the '+' button.
  5. Type 'systemconfiguration' in the search bar.
  6. Select 'SystemConfiguration.framework' from the list.
  7. Click on the 'Add' button.
  8. Type 'addsupport' in the search bar.
  9. Select 'AdSupport.framework' from the list.
  10. Click on the 'Add' button.
